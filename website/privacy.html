Privacy policy

A privacy tool's website should be auditable in one sitting. This one is. Draft note: this text is a mockup of the policy that will accompany the real site; nothing here is a binding statement yet.

01 / What this site collects

Nothing. This is a static site: no cookies, no analytics, no telemetry, no tracking pixels, no forms, no accounts. Fonts are served from this site's own folder — the pages trigger zero requests to third-party servers. Links to external sources in the dossier load only if you click them, and what happens then is between you and that site.

02 / What the host sees

Like any website, the server that delivers these files (e.g. a static-hosting provider) can log standard connection data such as your IP address and requested page. We don't receive, add to, or analyze such logs. If you consider that unacceptable, the entire site works identically from a local copy — clone the repository and open website/index.html offline.

03 / What the product will collect

The planned application follows the same rule, stated in the verification section: no telemetry, no accounts, redaction mappings held in session memory only. When the product ships, this policy will be replaced by a binding one — and the change will be visible in the site's public git history.
